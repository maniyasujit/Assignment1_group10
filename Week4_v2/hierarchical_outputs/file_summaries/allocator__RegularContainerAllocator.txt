**Key Functionality:**
This Java class, `RegularContainerAllocator`, is responsible for allocating containers to applications in a YARN (Yet Another Resource Negotiator) cluster, considering factors like locality, label, and resource availability. It implements a delayed scheduling mechanism to improve locality allocation.

**Core Logic:**

1. **Pre-checks**: Verify if a pending resource request can be allocated to a given node or set of nodes, considering exclusivity, pending resources, node partitions, headroom, and placement constraints.
2. **Locality-based Allocation**: Attempt to allocate containers in order of locality preference (Node Local, Rack Local, Off-Switch) while respecting scheduling opportunities and delays.
3. **Resource Management**: Handle container allocation, reservation, and deallocation, ensuring resource limits and availability are respected.
