**Summary**

* **Key Functionality**: Calculates the available "headroom" resources (e.g., memory, vCores) for a given application in a YARN (Yet Another Resource Negotiator) capacity scheduler queue.
* **Core Logic**: 
	+ Retrieves current queue limits and cluster resources.
	+ Determines headroom based on application's requested partitions (if any); otherwise, uses queue's default headroom calculation.
	+ Aggregates headroom from multiple partitions (if requested).
	+ Handles edge case where calculated headroom exceeds limits (sets to 0).

	+ **Inputs**: `UsersManager.User`, `AbstractLeafQueue`, `FiCaSchedulerApp`, `QueueResourceLimitsInfo`
	+ **Outputs**: `Resource` object representing available headroom (memory, vCores)
